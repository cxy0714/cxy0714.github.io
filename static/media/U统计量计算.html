U统计量计算新算法

U统计量的定义

在统计理论中，UUU统计量一直是研究者的“心头好”。它最大的优势在于：能够轻松构造无偏估计，名字里的 UUU 其实就是 “unbiased” 的缩写。

不过，虽然原理简单，UUU统计量的计算却常常令人头疼。先来看mmm阶UUU统计量的一般形式：

U(h)=1n(n−1)(n−2)⋯(n−m−1)∑1≤i1≠i2≠⋯≠im≤nh(Xi1,Xi2,⋯ ,Xim),(1)\mathbb{U}(h) = \frac{1}{n(n-1)(n-2)\cdots(n-m-1)}\sum_{1 \le i_1 \neq i_2 \neq \cdots \neq i_m \le n} h(X_{i_1}, X_{i_2}, \cdots, X_{i_m}),\qquad\qquad (1)U(h)=n(n−1)(n−2)⋯(n−m−1)1​1≤i1​=i2​=⋯=im​≤n∑​h(Xi1​​,Xi2​​,⋯,Xim​​),(1)

其中，X1,X2,⋯ ,Xn∈Ω⊆RpX_1, X_2, \cdots, X_n \in \Omega \subseteq \mathbb{R}^{p}X1​,X2​,⋯,Xn​∈Ω⊆Rp 是 nnn 个独立同分布的样本；h:Ωm→Rh : \Omega^m \to \mathbb{R}h:Ωm→R 称为核函数；指标条件 i1≠i2≠⋯≠imi_1 \neq i_2 \neq \cdots \neq i_mi1​=i2​=⋯=im​ 表示取的mmm个样本互不相同因此互相独立。

UUU统计量之所以重要，在于它天然满足无偏性：

E[U(h)]=E[h(X1,X2,⋯ ,Xm)],\mathbb{E} [ \mathbb{U} (h)] = \mathbb{E} [h(X_{1}, X_{2}, \cdots, X_{m}) ],E[U(h)]=E[h(X1​,X2​,⋯,Xm​)],

其中 X1,X2,⋯ ,XmX_{1}, X_{2}, \cdots, X_{m}X1​,X2​,⋯,Xm​ 是相互独立同分布的随机变量。换句话说，UUU统计量就是把所有可能的互不相等的mmm元样本组都带入核函数 hhh 计算，再取平均值，从而得到一个对目标参数的无偏估计。

UUU统计量常常和它的“双胞胎”-VVV统计量一起提及，mmm阶VVV统计量的定义为：
V(h)=1nm∑1≤i1,i2,⋯ ,im≤nh(Xi1,Xi2,⋯ ,Xim).(2)\mathbb{V}(h) = \frac{1}{n^m}\sum_{1 \le i_1, i_2,\cdots, i_m \le n} h(X_{i_1}, X_{i_2}, \cdots, X_{i_m}).\qquad\qquad (2)V(h)=nm1​1≤i1​,i2​,⋯,im​≤n∑​h(Xi1​​,Xi2​​,⋯,Xim​​).(2)

注意UUU和VVV的区别就在于计算中指标是否存在限制，VVV统计量的计算中对指标(i1,i2,⋯ ,im)(i_1, i_2,\cdots, i_m)(i1​,i2​,⋯,im​)没有任何限制，因此总共有nmn^mnm个值。

很容易可以看出来，VVV统计量可以表示成一组UUU统计量的线性组合（就是加加减减乘乘），反过来UUU统计量也可以表示成一组VVV统计量的线性组合（例子在稍后展示）。所以只要有一方可以高效计算，另一方就可以通过线性组合得到，我们算法就是这样子做！

计算难题

回到公式 (1)。如果直接用 for 循环枚举mmm阶组合来计算，那么复杂度是 O(nm)O(n^m)O(nm)（假设hhh的计算复杂度与nnn无关）。 作为对比，矩阵乘法或矩阵求逆的复杂度是 O(n3)O(n^3)O(n3)，因此，一旦阶数 m>3m > 3m>3，UUU统计量的计算往往会变得难以承受。

而偏偏，我们关注的统计量就是高阶UUU统计量。James M. Robins 等人在 2008 年提出的 高阶影响函数（Higher Order Influence Function, HOIF），在各种假设下均能达到最优估计速率。HOIF 是一个高阶 UUU统计量，最优阶数大约是 m∼log⁡(n)m \sim \sqrt{\log(n)}m∼log(n)​。这意味着，HOIF 的实际应用必须直面高阶 UUU 统计量的计算挑战。

m (m≥2m \ge 2m≥2) 阶HOIF的计算可以化简为核函数为{hjHOIF}j=2m\{ h^{HOIF}_{j} \}_{j=2}^{m}{hjHOIF​}j=2m​的UUU统计量的线性组合，核函数hjHOIFh^{HOIF}_{j}hjHOIF​可以简化为以下形式:

更进一步，我们发现这套方法并不局限于 HOIF对应的这一类UUU统计量，而是可以推广到任意 UUU 统计量。借助图论工具，我们能够准确刻画出该算法的“最乐观”复杂度上界。
更妙的是，实际实现完全可以依赖Numpy和PyTorch 提供的 Einsum 函数，天然支持 GPU 和 CPU 并行，从而在工程上也十分高效。

新算法

接下来我们介绍新算法的实现，这依赖于Python库中的强大函数Nupmy.einsum 和 torch.einsum。他们的底层实现已经高度优化，torch版本的einsum可以便捷地使用cpu和gpu并行。

Einsum

Einsum函数实现的是什么呢？网上已经很多帖子（比如）介绍了Einsum，我们的论文中也给了一个更严格的数学定义。简单来说，Einsum是对一系列张量的操作，最主要的功能是输入一列张量，对部分或者全部指标做无约束的求和，最终得到一个张量或者一个常数。

Einsum 调用格式 对应结果
np.einsum('ij,jk ->ik', A, B) Dik=∑jAijBjkD_{ik} = \sum_j A_{ij} B_{jk}Dik​=∑j​Aij​Bjk​
np.einsum('ijk->i', X) Di=∑j,kXijkD_i = \sum_{j,k} X_{ijk}Di​=∑j,k​Xijk​
np.einsum('ij,jk,kl-> ', A, B,C) D=∑i,j,k,lAijBjkCklD = \sum_{i,j,k,l} A_{ij} B_{jk} C_{kl}D=∑i,j,k,l​Aij​Bjk​Ckl​
np.einsum('ijk,jjk,kkl-> ', A, B,C) D=∑i,j,k,lAijkBjjkCkklD = \sum_{i,j,k,l} A_{ijk} B_{jjk} C_{kkl}D=∑i,j,k,l​Aijk​Bjjk​Ckkl​

像'ij,j -> j'这样的式子叫做einsum表达式，"->"左侧代表了输入张量的指标分布，"->"右侧代表最终的张量剩下来的指标（如果为0，就代表结果为常数），这本来Einstein老爷子（提出该记号时正年轻，1916年）为了方便写公式约定的记号，左侧是全部的指标的分布，右侧是剩余的指标，左侧中出现右侧不出现的指标就是求和求掉了。矩阵的所有运算都可以用Einsum来表示，当需要用3阶以上的张量运算时，Einsum就成了最有效的工具。
